# HausmanHub 1.9.3: пробное внутреннее управление одной комнатой

Дата: 2026-07-20. Пункт плана: 34.

## Что сделано

HausmanHub впервые может сам физически управлять климатом — ровно одной
явно настроенной пробной комнатой и только при согласии всех защит. Раз в
минуту отдельный контурный тик проверяет цепочку:

1. настроенная пробная комната (`climate_canary_room_id`);
2. режим моста `canary`;
3. автоматический контур;
4. свежее наблюдение;
5. готовая комната без локальных отказов;
6. понятный результат сравнения (не «нельзя сравнить»);
7. каждое устройство — пробного объёма `canary` с управляющей привязкой
   HausmanHub и нужными возможностями;
8. полный перевод плана без ограничений.

Действие возможно только при расхождении внутреннего плана с наблюдаемым
состоянием. Совпадение честно пропускается как «уже актуально». Любая
неопределённость закрывает управление без единого вызова. Выполняются
только строгие вызовы адаптеров из закрытого перечня, по порядку, с
остановкой при первой ошибке.

Квитанция каждой проверки строго редактирована: постоянный код комнаты,
понятный статус (применено, уже актуально, отклонено, ошибка), ограниченные
причины и счётчики вызовов. Объекты, полезные нагрузки и ответы сервисов в
неё не попадают.

## Безопасность границы

Скелетная проверка репозитория теперь фиксирует: вызовы Home Assistant
возможны только в исполнителе пробных вызовов (`climate_ha_executor.py`) и
прежнем canary-переключателе (`switch.py`). Остальные модули не могут
обратиться к сервисам — это останавливает сборку. Пробную комнату оператор
обязан убрать из комнат прежнего климатического модуля: двойное управление
одними устройствами недопустимо. Прежний модуль и Android-проект не
изменялись и использовались только для чтения.

## Проверки

Отдельно проверены применение одной расходящейся пробной комнаты, пропуск
«уже актуально», отказ вне режима `canary`, устройство вне пробного объёма,
неполный перевод, отказ сервиса с частичным выполнением, отсутствие
исполнителя, отображение строгих вызовов в сервисные данные Home Assistant
с остановкой при первой ошибке и строгость модели решения и квитанции.

Подготовленное дерево прошло 513 локальных тестов, проверку пакета HACS,
названия продукта, границ репозитория и совместимости с Android-моделями, а
также отдельные проверки на чистых Home Assistant Core 2026.6.4 и 2026.7.0.
