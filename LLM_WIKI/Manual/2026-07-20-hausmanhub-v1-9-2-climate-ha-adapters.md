# HausmanHub 1.9.2: строгие адаптеры устройств Home Assistant

Дата: 2026-07-20. Пункт плана: 33.

## Что сделано

Внутри HausmanHub созданы строгие адаптеры устройств Home Assistant.
Проверенный внутренний план каждого устройства переводится в точный список
вызовов из закрытого перечня сервисов:

- `climate.set_hvac_mode` — только режимы `cool`, `heat`, `off`;
- `climate.set_temperature` — только 10–35 °C;
- `climate.set_fan_mode` — только известные ступени;
- `humidifier.turn_on` / `humidifier.turn_off` — только питание увлажнителя.

Каждый вызов указывает один проверенный объект из приватной привязки реестра.
Произвольных полей, чужих привязок и сырых значений нет. Кондиционер
получает режим, температуру и вентилятор в фиксированном порядке;
термоголовка — только температуру; увлажнитель — только питание; тёплый
пол — только температуру и выключение. Вызов смены режима требует заявленной
возможности `HVAC_MODE`.

Попутно исправлена политика тёплого пола: в отопительном сезоне она выдаёт
понятное действие «задать температуру». Раньше слой оборудования отдавал
действие `heat` с температурой, которое не проходит строгую проверку
финального плана, и комната с тёплым полом оставалась без политики из-за
внутренней ошибки расчёта. Эталонные сценарии это не затрагивает — среди
них нет тёплого пола. Независимый просмотр Kimi k3 сначала остановил набор
(FAIL): несогласованный вызов смены режима для тёплого пола и отсутствие
проверки `HVAC_MODE`. После одной итерации исправлений с новыми тестами
оба замечания закрыты.

Перевод останавливается честно и с понятной причиной: нет управляющего
объекта, нет нужной возможности (отсутствие возможности вентилятора блокирует
весь перевод устройства, а не сбрасывает часть плана), действие не
поддерживается, режим наблюдения или удержания, нечего переводить. Тихий
режим отмечается как непереведённый — у него нет строгого вызова.

## Безопасность границы

Это только перевод. Свойство `commands_enabled` всегда равно `False`: ни
Home Assistant, ни прежний климатический модуль вызовов не получают.
Управляемые модулем устройства без привязки HausmanHub дают ноль вызовов с
причиной `missing_control_endpoint`. Прежний климатический модуль и
Android-проект не изменялись и использовались только для чтения.

## Проверки

Отдельно проверены перевод безопасной остановки, полный перевод охлаждения в
фиксированном порядке, устройство без привязки, блокировка при пропавшей
возможности вентилятора и при отсутствии `HVAC_MODE`, увлажнитель,
термоголовка, тёплый пол, строгость модели вызовов (чужие значения, пределы,
формат объекта, поддельная версия) и отсутствие приватных полей в
сериализации.

Подготовленное дерево прошло 504 локальных теста, проверку пакета HACS,
названия продукта, границ репозитория и совместимости с Android-моделями, а
также отдельные проверки на чистых Home Assistant Core 2026.6.4 и 2026.7.0.
