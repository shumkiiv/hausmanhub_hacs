# HausmanHub 1.9.7: внутренняя цепочка на собственном наблюдении

Дата: 2026-07-21. Пункт плана: 36, подшаг 36c.

## Что сделано

Внутренняя климатическая цепочка HausmanHub больше не зависит от внешнего
Climate API. Предпросмотр, цели, потребности, разрешения, оборудование,
стабильность, политика, изоляция отказов, сравнение, перевод вызовов,
пробная комната и управляемые комнаты читают только собственное наблюдение
Home Assistant (адаптер версии 1.9.6, привязки реестра версии 2).

Новая внешняя граница `climate_ha_state_view.py`
(`HomeAssistantClimateStateView`) превращает состояния Home Assistant в
ограниченные неизменяемые значения: строгий перечень атрибутов
(`hvac_action`, `temperature`, `current_temperature`, `fan_mode`,
`humidity`), ограниченная длина состояния, время последнего обновления в
миллисекундах. Скелетная проверка исполнительной границы не нарушена:
чтение состояний разрешено, вызовы служб остаются только в двух типовых
исполнителях.

## Правила перехода

- Нативное наблюдение — единственный вход внутренней цепочки. Факты двух
  систем не смешиваются ни покомнатно, ни по атрибутам.
- Сломанный источник состояний закрывает наблюдение (недоступный снимок)
  без возврата к мосту.
- Режим `disabled` не наблюдает вовсе, вид состояний не опрашивается.
- Сравнение решений теперь сверяет собственный план с фактическим
  состоянием Home Assistant: совпадение подавляет лишние вызовы,
  расхождение разрешает действие, несравнимость закрывает управление.

## Что не изменилось

Внешний Climate API пока обслуживает публичный снимок Android, применение
настроек (apply/расписание/временная температура), готовность и мастера
настройки. Их перевод — подшаги 36d–36f. Работающий климатический модуль
и Android-приложение не изменялись.

## Проверки

- 549 локальных тестов, включая новый набор: цепочка без единого обращения
  к мосту, минутный тик исполняет расхождение из состояний Home Assistant
  с «ядовитым» мостом, режим `disabled` игнорирует вид состояний, сломанный
  вид закрывает наблюдение, граница состояний ограничивает атрибуты.
- `tools/check_local_release.py` — успешно.
- Одноразовые Home Assistant Core 2026.6.4 и 2026.7.0 — успешно.
- Независимый просмотр Kimi k3 — вердикт записывается после просмотра.
