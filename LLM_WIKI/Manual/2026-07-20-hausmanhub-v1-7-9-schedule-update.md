# HausmanHub 1.7.9: настройка расписания через API

Дата: 2026-07-20. Пункт плана: 19.

## Результат

Локальный администратор Home Assistant может сохранить время начала профилей
**«День»** и **«Ночь»**, включить автоматическое переключение или безопасно
выключить его через:

`POST /api/hausman_hub/v1/admin/climate-schedule`

Запрос и ответ имеют отдельные строгие контракты версии 1. Запрос связан с
полной сохранённой настройкой через `setup_revision`; устаревшее значение
отклоняется HTTP 409 до записи.

## Правила безопасности

- Включение разрешено только для контура в режиме `automatic`, при мосте
  `managed` и с явным `confirm_automatic_application: true`.
- Выключение разрешено при любом режиме моста, включая `canary`, чтобы
  автоматическое управление всегда можно было остановить.
- Время принимается только в формате `ЧЧ:ММ`; начало дня и ночи не совпадает.
- Сохранение делает одну запись в хранилище контура, не читает внешний Climate
  API и не отправляет команд устройствам.
- Память runtime меняется только после успешной записи.
- Комнаты, устройства, профили, активный профиль, режим и имя контура
  сохраняются без изменений.
- При изменении времени или выключении временные температуры очищаются, потому
  что прежняя граница их окончания больше недействительна. Неизменённое
  включённое расписание сохраняет временные значения.

## Честный ответ

Квитанция всегда сообщает `commands_sent: false`, новое `setup_revision`,
сохранённое расписание и число очищенных временных значений. Поле
`automatic_application_pending` истинно только когда полное управляемое
расписание действительно должно применить профиль на ближайшей обычной
проверке.
