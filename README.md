# HausMan Hub HASC

HASC — интеграция Home Assistant и будущая единая система автоматических
контуров дома. Климат — первый контур. Позднее на той же основе можно добавить
освещение, воду, безопасность, энергию и другие виды автоматизации.

Текущая версия — **1.0.0**.

## Как теперь устроен климат

Обычная настройка выглядит так:

1. Открыть **HASC → Настроить → Автоматические контуры**.
2. Выбрать **Настроить климатический контур**.
3. Один раз указать локальный адрес уже работающего климатического контура.
4. Отметить комнаты и находящиеся в них кондиционеры, термоголовки,
   увлажнители, тёплый пол и датчики.
5. Задать комфортную температуру, влажность и характер работы.
6. Проверить понятное резюме и сохранить контур.

Внутренние идентификаторы устройств, Home Assistant entities, `source_id`,
bridge, shadow и canary в обычном мастере не показываются. Технические средства
миграции и диагностики находятся в разделе **Дополнительные настройки**.

## Что делает версия 1.0.0

- Добавляет общую версионную модель автоматических контуров. Климат — первый
  поддерживаемый тип.
- Создаёт климатический контур сразу для нескольких выбранных комнат и
  устройств.
- Автоматически формирует стабильные внутренние связи HASC по данным уже
  работающего климатического двигателя.
- Хранит понятные параметры комфорта: температуру 18–28 °C с шагом 0,5 °C,
  влажность 30–70 % с шагом 5 % и мягкую, обычную или быструю стратегию.
- Использует существующий `hausman-climate` как климатический алгоритм. В нём
  остаются проверенные профили, задержки, защита от частых переключений,
  ручные режимы, контроль доступности и подтверждение работы устройств.
- Показывает «автоматически» только когда существующий двигатель действительно
  находится в авто-режиме, имеет право управления, получает свежие данные,
  видит доступные устройства и использует совпадающие цели.
- Отдаёт приложению планшета отдельный локальный контракт
  `GET /api/hausman_hub/v1/contours` без приватных идентификаторов.
- Сохраняет список устройств и описание контура атомарно: неполная настройка не
  заменяет рабочую.

## Важная граница 1.0.0

Версия 1.0.0 объединяет существующий климатический алгоритм под понятной
моделью HASC, но ещё не переносит его внутрь интеграции. HASC сохраняет желаемые
параметры и проверяет их совпадение с двигателем. Если они отличаются, статус
будет **«нужно проверить настройки»**. Эта версия сама не переписывает параметры
двигателя и не отправляет климатические команды.

Поэтому для первого включения выбирайте значения, которые уже используются
существующим климатическим контуром. Следующий этап 1.x — безопасная
синхронизация целей и режимов из HASC с квитанцией результата, затем управление
ими с Android-планшета.

Отключение контура в HASC прекращает его представление как активного контура
HASC, но не выключает уже работающий внешний климатический двигатель и не
отправляет команды устройствам.

## Установка через HACS

1. В HACS добавьте пользовательский репозиторий
   `https://github.com/shumkiiv/hausmanhub_hasc` типа **Интеграция**.
2. Установите **HausMan Hub HASC**.
3. Перезапустите Home Assistant.
4. Откройте **Настройки → Устройства и службы → Добавить интеграцию** и
   добавьте **HausMan Hub HASC**.
5. Для обычной работы оставьте первоначальный режим
   **Обычное чтение сведений**.

Установка и обновление сами по себе не настраивают живой климат и не дают HASC
права отправлять команды.

## Разделы настроек

- **Автоматические контуры** — основное место настройки и проверки климата.
- **Общие сведения HASC** — девять обезличенных диагностических счётчиков и
  частота их обновления; на климат не влияет.
- **Дополнительные настройки** — старые средства подключения, реестр,
  предпросмотр и служебный тест. Для обычной настройки они не нужны.

Подробная схема описана в [руководстве по климатическому
контуру](docs/climate-control-architecture.md), а проверка установки — в
[инструкции для Home Assistant](docs/home-assistant-safe-check.md).

## Безопасность

HASC не хранит в репозитории ключи, пароли, адрес дома, реальные идентификаторы
устройств или снимки настроек. Контур принимает только локальный HTTP(S)-адрес
с буквальным частным IP. Публичные адреса, перенаправления, имена сайтов, пути,
параметры, логины и пароли отклоняются.

Приложение планшета не может передать произвольный сервис Home Assistant,
внутренний идентификатор устройства, адрес или команду нижнего уровня.

## Для разработчиков

Полная локальная проверка:

```shell
python3 tools/check_local_release.py
```

Она использует синтетические данные, не подключается к дому и не отправляет
команды устройствам. Правила разработки описаны в
[CONTRIBUTING.md](CONTRIBUTING.md) и
[инженерных требованиях](docs/engineering-standards.md).

Kimi должен проверить окончательный текущий набор изменений до того, как
изменение будет считаться завершённым или будут выполнены коммит, отправка,
выпуск, развёртывание или публикация.

Если Kimi временно недоступен, другой независимый просмотр может поддержать
работу. Он позволяет продолжать любые изменения внутри границ HASC: код, тесты,
документацию, местные проверки и исправления. Он не разрешает коммит, отправку,
выпуск, развёртывание, публикацию или новые права.

Исключение для изменения только документации действует лишь когда в наборе нет
кода; в смешанном наборе действует финальная проверка Kimi.
