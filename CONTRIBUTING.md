# Как безопасно улучшать HASC

HASC — отдельный проект для безопасного просмотра и проверки. Он не управляет
домом.

## Что можно предлагать

- улучшения местных проверок и понятных инструкций;
- исправления, которые не дают HASC новых прав;
- работу только с учебными примерами данных;
- улучшения двух безопасных режимов: «Только чтение» и «Проверка без
  изменений».

## Что нельзя присылать

- ключи, пароли, адрес дома, названия устройств или их внутренние номера;
- снимки настроек Home Assistant или Node-RED;
- команды устройствам, способы запуска команд или файлы для установки на дом;
- изменение правил климата, бытовых сценариев или центра управления домом.

## Порядок работы

1. Сначала прочитайте [границы проекта](docs/architecture-traceability.md).
2. Сделайте небольшое изменение с одной понятной целью.
3. Перед отправкой запустите `python3 tools/check_local_release.py`.
4. Для изменения кода получите независимый просмотр Kimi и устраните замечания.
5. В описании изменения кратко напишите, что изменилось и какая проверка
   пройдена. Не прикладывайте данные дома, даже если кажется, что они полезны
   для объяснения ошибки.

Если предложению нужны новые права, связь с домом или действие над устройством,
не начинайте работу: сначала нужно отдельное решение владельца.
