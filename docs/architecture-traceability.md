# Как HASC соблюдает исходные правила дома

Этот документ — короткая карта безопасности. Исходные архитектурные документы
живут в боевом репозитории HausMan Hub и читаются здесь только как источник
правил. HASC не копирует оттуда настройки, адреса, названия устройств или
команды.

## Простая картина

HASC сейчас — отдельное приложение для безопасного просмотра и первого
ограниченного canary-управления. Оно может переключать только один явно
выбранный виртуальный `input_boolean`; физическими устройствами оно не
управляет. Боевой проект HausMan Hub остаётся тем местом, где живут климат,
бытовые правила и сам центр управления домом.

| Исходное правило | Что это означает для HASC сейчас | Чем это проверяется здесь |
| --- | --- | --- |
| Разделение контуров и ADR | HASC не владеет правилами климата, бытовыми сценариями, общим описанием дома или центром управления. | Границы описаны в [read-only skeleton](read-only-skeleton.md) и проверяются местными тестами. |
| Общий контракт дома | HASC может проверять только учебные примеры общего описания, но не меняет его и не получает настоящие данные. | `fixtures/common_contract/` и местная проверка. |
| Реестр автоматизаций | HASC не создаёт и не меняет бытовые сценарии. | Нет файлов сценариев, команд или развёртывания; это проверяет проверка файлов. |
| Граница климата | HASC не меняет температуру, режимы, паузы или правила климата. | Canary принимает только домен `input_boolean`; climate и другие физические домены отклоняются. |
| Роль центра управления домом | HASC не принимает решения за другие части дома. | Два режима наблюдения дополнены только одним явно включаемым helper-canary без устройства. |
| HACS-проверка | Установка через HACS сама не включает управление. | Canary выключен по умолчанию и требует отдельного выбора одного helper. |
| Проверки общего контракта | Проверяются только учебные данные без настоящих названий и адресов. | Три учебные проверки запускаются одной местной командой. |
| Настройки интеграции | Режимы остаются «Только чтение» и «Проверка без изменений»; управление включается отдельным canary-флагом. | Форма отклоняет другой режим, небулевый флаг и любую цель кроме одного `input_boolean`. |
| Диагностика и подсказки | Отчёт содержит только разрешённые общие сведения и статус canary, но не его цель; подсказки ничего не исправляют сами. | Тесты фиксируют allow-list и отсутствие runtime-идентификатора. |
| Сравнение без изменений | Сравнение остаётся недоказанным, пока владельцы контуров не примут отдельное решение. | Учебный пример помечает результат как неразрешённый. |
| Прямое выполнение | Общая власть над устройствами заблокирована; разрешён только helper-canary. | Перед каждой командой проверяются одна запись, флаг, точная цель, домен и доступность helper. |

## Что можно делать без нового решения

- улучшать местные учебные проверки и понятные инструкции;
- проверять безопасность установки, удаления и обновления HASC в пустой
  учебной среде;
- показывать только девять заранее разрешённых общих чисел о доме;
- улучшать уже одобренный canary только внутри одного `input_boolean` и его
  проверенного отката;
- исправлять ошибки, которые не расширяют доступ HASC.

## Что нельзя начинать автоматически

- передавать команды физическим устройствам или через существующие части дома;
- расширять canary на другой домен или несколько целей;
- менять климатические правила, бытовые сценарии или правила центра дома;
- добавлять настоящие названия, адреса, ключи, снимки потоков или команды в
  этот репозиторий.

Для любого физического домена нужно отдельное решение владельца. Потребуются
доказанное сравнение без изменений, конкретная цель, план безопасного
включения, аварийная остановка и возврат назад, а также подпись владельца
нужной части дома.
